# Hybrid Search 規則調參表

這份表把目前 `core/filters.py` 的實際規則整理成「高信心 / 邊界 / 排除」三層，方便後續維護與調參。

## 1. 教育機構判定

### 會視為教育單位

| 類別 | 字詞 |
|---|---|
| 大專校院 | `大學`、`學院`、`科技大學`、`技術學院`、`專科學校`、`專科` |
| 軍校 | `軍官學校`、`國防大學`、`國防醫學院` |
| 中小學 | `學校`、`國小`、`國中`、`中學`、`高中`、`高職` |
| 教育行政 | `教育局`、`教育處` |

### 直接排除

| 類別 | 字詞 | 說明 |
|---|---|---|
| 強排除 | `附設醫院`、`附屬醫院`、`分院`、`診所` | 一律排除 |
| 弱排除 | `醫院` | 若機關名稱含 `醫學院` 可放行 |

### 白名單

| 字詞 | 用途 |
|---|---|
| `醫學院` | 覆蓋 `醫院` 的弱排除，保留教學單位 |

## 2. 高信心規則

以下字詞命中後，通常會直接進入高信心候選，不需要靠 embedding 補回。

### 核心詞群

| 類別 | 代表字詞 |
|---|---|
| AI / 資料分析 | `ai`、`人工智慧`、`機器學習`、`深度學習`、`大數據`、`資料分析`、`數據分析`、`資料科學` |
| 系統 / 平台 | `系統`、`平台`、`資訊系統`、`應用系統`、`管理系統`、`校務系統`、`教務系統`、`學務系統`、`公文系統` |
| 資安 | `資安`、`防火牆`、`waf`、`入侵`、`弱點`、`資通安全`、`防護`、`防毒`、`零信任`、`ids`、`ips`、`滲透測試`、`弱點掃描`、`資安監控`、`soc`、`加密`、`認證`、`網頁應用程式防火牆` |
| 網路 | `網路`、`交換器`、`基地台`、`ap`、`無線`、`wifi`、`網通`、`通訊設備`、`路由器`、`無線網路`、`網路設備`、`光纖`、`網卡`、`乙太網路`、`區域網路` |
| 文件 / 檔案 | `文件管理`、`檔案管理`、`電子公文`、`知識管理`、`文管`、`文書`、`公文系統` |
| 硬體 | `資訊設備`、`電腦設備`、`筆記型電腦`、`伺服器`、`個人電腦`、`pc`、`主機`、`工作站`、`掃描器`、`螢幕` |
| 軟體 / 授權 | `軟體`、`授權`、`訂閱`、`軟體訂閱`、`應用程式`、`app`、`應用軟體`、`作業系統`、`office`、`防毒軟體` |
| 雲端 / 虛擬化 | `雲端`、`cloud`、`saas`、`iaas`、`paas`、`虛擬化`、`vm`、`容器`、`docker`、`私有雲`、`公有雲`、`混合雲` |
| 儲存 / 備份 | `儲存`、`備份`、`nas`、`san`、`磁碟陣列`、`資料庫`、`database`、`db`、`sql`、`儲存設備`、`備份系統` |

### 高信心門檻

- `HIGH_CONFIDENCE_THRESHOLD = 2`
- 核心詞群命中後，分數通常會直接達標
- 如果同時出現 `系統 / 平台 / 資訊系統 / 應用系統` 與 `管理 / 整合 / 導入 / 建置 / 開發`，通常會直接進高信心

## 3. 邊界規則

這些詞不建議單獨視為強命中，適合留給 embedding 補語意。

| 類別 | 字詞 | 建議用途 |
|---|---|---|
| 邊界詞 | `智慧`、`協作`、`整合`、`導入`、`建置`、`開發`、`文件`、`檔案` | 只加弱分，用於灰區案件 |
| 上下文詞 | `管理` | 只有搭配 `系統`、`平台`、`資訊系統`、`文件管理`、`檔案管理`、`公文系統` 才加分 |
| 上下文詞 | `文件`、`檔案` | 只有搭配 `管理`、`系統`、`電子公文`、`文書`、`公文` 才加分 |
| 上下文詞 | `教學`、`校園`、`學習`、`教室` | 搭配 `平台`、`系統`、`智慧` 時才加分 |
| 上下文詞 | `協作` | 搭配 `平台`、`系統`、`管理` 時才加分 |
| 上下文詞 | `整合` | 搭配 `平台`、`系統`、`資訊系統`、`應用系統`、`雲端` 時才加分 |
| 上下文詞 | `導入`、`建置` | 搭配 `平台`、`系統`、`雲端`、`資訊系統` 時才加分 |
| 上下文詞 | `開發` | 搭配 `平台`、`系統`、`文件`、`檔案`、`公文` 時才加分 |

### 邊界門檻

- `LOW_CONFIDENCE_THRESHOLD = 0`
- 分數大於 0 但未達高信心者，保留為 boundary 候選
- 這些案件會交給 embedding recall 做語意補強

## 5. Embedding 補強規則

embedding 只處理邊界候選，不做第一層主篩選。

### 目前建議配置

- `EMBEDDING_MODEL = paraphrase-multilingual-MiniLM-L12-v2`
- `EMBEDDING_TOP_K = 30`
- `EMBEDDING_SIMILARITY_THRESHOLD = 0.62`

### 適合交給 embedding 的灰區例子

- `智慧教室`
- `教學平台`
- `文件流轉`
- `校務系統`
- `校園協作平台`
- `電子公文流程`

## 6. 維護建議

### 先加分、再觀察

- 新字詞先放進邊界層
- 觀察 1-2 次實際通知結果後，再決定是否升級成高信心

### 若誤報增加

- 先檢查是否被 `SYSTEM_TERMS` 或 `DOCUMENT_TERMS` 過度放行
- 再檢查是否有新的排除詞需要補進 `EXCLUDE_KEYWORDS`

### 若漏報增加

- 先看是不是邊界詞不夠多
- 再看 embedding 的 `SIMILARITY_THRESHOLD` 是否需要微調

### 不建議直接升級成高信心的詞

- `管理`
- `整合`
- `導入`
- `建置`
- `開發`
- `文件`
- `檔案`

這些詞很常出現在資訊標案，但也容易誤傷其他類型，建議維持在邊界層。
